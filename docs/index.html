How it works

tinker-web is a tiny standalone PHP tool — no runtime dependencies. It talks to the target app through a fresh runner process, so it works with any project and clashes with none.

01

Run it

tinker-web ~/code/app starts a local server on 127.0.0.1 and opens your browser.

02

Write PHP

Type and press ⌘/Ctrl+↵ — or flip on Auto-run to evaluate live as you finish each statement.

03

Runner boots the target

A fresh subprocess requires the project's autoloader, boots that app, and evaluates your snippet in its context.

04

Rendered back

Each statement's output and value come back as collapsible VarDumper HTML — dump()/dd() included, and dd() stops the run cleanly.

Features

◆Works with any project

Point it at any local Laravel app by path. The target needs nothing installed — the runner bootstraps it and uses its own PsySH/Tinker if present, with a plain-eval fallback otherwise.

◆Collapsible rich dumps

Return values and dump()/dd() render as VarDumper's interactive, collapsible HTML with Tinker casters — click to expand Eloquent models and arrays. dd()/exit()/die() stop the run cleanly with a marker; echoed output is captured separately.

◆Live auto-run & notebook

Opt into Auto-run to re-evaluate as you finish each statement. Snippets run as a notebook — each top-level statement gets its own result cell, and state persists within a run.

◆Local & locked down

Binds 127.0.0.1 only, requires a random per-session token, and enforces a Host-header allowlist against DNS-rebinding. Your machine, your apps.

◆Connection manager

Switch between projects from the toolbar; recent projects are remembered. One running tool, many apps.

◆Process-isolated

Each run is a fresh subprocess rooted in the target — clean state every time, and a crashing eval can never take the server down.

◆Zero runtime deps

Pure PHP 8.2+ core (stream_socket_server + proc_open). Fast to install, nothing to conflict with.

Install & run

Requires PHP 8.2+. A local Laravel app to target (any version).

# global install — then ensure your composer global bin is on PATH
composer global require amims71/tinker-web

# or clone and run directly
git clone https://github.com/amims71/tinker-web && cd tinker-web && composer install
php bin/tinker-web /path/to/app

# usage
tinker-web                 # target the current directory
tinker-web /path/to/app    # target a specific project
tinker-web --port=9000     # fixed port (default: ephemeral)
tinker-web --no-open       # don't auto-open the browser

Security

tinker-web executes arbitrary PHP against the target app, so it is deliberately local-only. Run it on your own machine, against apps you trust.

127.0.0.1Binds to loopback exclusively — never 0.0.0.0.
tokenA random per-session token in the URL, compared in constant time.
hostA Host-header allowlist (127.0.0.1 / localhost) blunts DNS-rebinding.
